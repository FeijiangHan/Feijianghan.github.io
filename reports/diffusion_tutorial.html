七个算法，一个故事

在深入每个算法之前，先建立全局视野——理解它们按什么逻辑依次出现，以及每个解决了上一个的哪个瓶颈。

如果你只学过 LLM/VLM，理解扩散模型需要一次思维范式的切换。自回归语言模型把概率分解为逐 token 的条件乘积，每步生成下一个词。扩散模型则完全不同：它把数据分布 \(p(x)\) 理解为一个连续传输过程的终点——从简单高斯噪声 \(\mathcal{N}(0,I)\) 出发，经过若干步去噪，最终落在真实数据分布上。

七个算法形成一条清晰的进化链，每一步解决上一步的核心瓶颈。

算法一 · Ho et al. 2020 [1]

DDPM 基础

去噪扩散概率模型——整个扩散家族的数学地基。理解它，就掌握了后续所有方法的骨架。

1.1 核心直觉

取一张清晰照片，每次往上叠加一点点高斯噪声，重复 1000 次后，照片变成纯静态。这个过程完全固定，不需要学习，称为前向过程。

反方向看：若神经网络能学会去掉一小步的噪声，那从纯噪声出发、一步步去噪，最终就能生成一张全新的真实图像。这个需要学习的过程称为反向过程。

与 LLM 对比理解：LLM 逐 token 生成，每步调用模型一次；DDPM 逐步去噪，生成一张图需要调用约 1000 次。两者都把复杂分布的学习分解为一系列更简单的条件预测问题。

1.2 前向过程：加噪的封闭公式

设原始图像为 \(\mathbf{x}_0\)，定义方差调度表 \(\beta_1 < \beta_2 < \cdots < \beta_T\)，每个 \(\beta_t \in (0,1)\)（例如从 \(0.0001\) 线性增到 \(0.02\)，总步数 \(T=1000\)）。每步前向过程只做一件事：

前向单步 $$q(\mathbf{x}_t \mid \mathbf{x}_{t-1}) = \mathcal{N}\!\left(\mathbf{x}_t;\, \sqrt{1-\beta_t}\,\mathbf{x}_{t-1},\; \beta_t \mathbf{I}\right)$$

将上一步稍微"收缩"（乘以 \(\sqrt{1-\beta_t}<1\)），再叠加方差为 \(\beta_t\) 的高斯噪声。

关键问题：要得到第 500 步的加噪图像，难道要串行累乘 500 次？不需要。定义 \(\alpha_t = 1-\beta_t\) 和 \(\bar{\alpha}_t = \prod_{i=1}^t \alpha_i\)，利用高斯合并性质（两个独立高斯之和仍是高斯）归纳推导，即可得到一步跳转公式。

1

两步高斯合并示例：设 \(\mathbf{x}_{t} = \sqrt{\alpha_2}\,\mathbf{x}_{t-1} + \sqrt{1-\alpha_2}\,\boldsymbol{\epsilon}_2\)，代入 \(\mathbf{x}_{t-1} = \sqrt{\alpha_1}\,\mathbf{x}_0 + \sqrt{1-\alpha_1}\,\boldsymbol{\epsilon}_1\)，展开得

$$\mathbf{x}_t = \sqrt{\alpha_1\alpha_2}\,\mathbf{x}_0 + \sqrt{\alpha_2(1-\alpha_1)}\,\boldsymbol{\epsilon}_1 + \sqrt{1-\alpha_2}\,\boldsymbol{\epsilon}_2$$
2

两个独立高斯 \(\mathcal{N}(0,\sigma_1^2\mathbf{I})\) 和 \(\mathcal{N}(0,\sigma_2^2\mathbf{I})\) 之和等于 \(\mathcal{N}(0,(\sigma_1^2+\sigma_2^2)\mathbf{I})\)。后两项的总方差为 \(\alpha_2(1-\alpha_1)+(1-\alpha_2)=1-\alpha_1\alpha_2\)，因此合并为单个噪声 \(\sqrt{1-\alpha_1\alpha_2}\,\boldsymbol{\epsilon}\)。

3

对任意 \(t\) 步归纳，得到核心封闭公式：

任意步加噪（核心） $$\mathbf{x}_t = \sqrt{\bar{\alpha}_t}\,\mathbf{x}_0 + \sqrt{1-\bar{\alpha}_t}\,\boldsymbol{\epsilon}, \quad \boldsymbol{\epsilon} \sim \mathcal{N}(0,\mathbf{I})$$ $$\text{即} \quad q(\mathbf{x}_t \mid \mathbf{x}_0) = \mathcal{N}\!\left(\mathbf{x}_t;\, \sqrt{\bar{\alpha}_t}\,\mathbf{x}_0,\; (1-\bar{\alpha}_t)\mathbf{I}\right)$$

给定原图和时间步 \(t\)，可一次采样得到对应的加噪图。\(t\to T\) 时 \(\bar{\alpha}_T\approx 0\)，图像趋近纯高斯噪声；\(t=0\) 时 \(\bar{\alpha}_0=1\)，图像不变。

1.3 反向过程与贝叶斯后验

目标是学习 \(p_\theta(\mathbf{x}_{t-1}\mid\mathbf{x}_t)\)。如果已知 \(\mathbf{x}_0\)，后验 \(q(\mathbf{x}_{t-1}\mid\mathbf{x}_t,\mathbf{x}_0)\) 可由贝叶斯公式解析计算：

真实后验（可解析） $$q(\mathbf{x}_{t-1}\mid\mathbf{x}_t,\mathbf{x}_0) = \mathcal{N}\!\left(\mathbf{x}_{t-1};\, \tilde{\boldsymbol{\mu}}_t(\mathbf{x}_t,\mathbf{x}_0),\; \tilde{\beta}_t\mathbf{I}\right)$$ $$\tilde{\beta}_t = \frac{1-\bar{\alpha}_{t-1}}{1-\bar{\alpha}_t}\,\beta_t$$ $$\tilde{\boldsymbol{\mu}}_t = \frac{\sqrt{\alpha_t}(1-\bar{\alpha}_{t-1})}{1-\bar{\alpha}_t}\,\mathbf{x}_t + \frac{\sqrt{\bar{\alpha}_{t-1}}\,\beta_t}{1-\bar{\alpha}_t}\,\mathbf{x}_0$$

推导过程：将 \(q(\mathbf{x}_{t-1}\mid\mathbf{x}_t,\mathbf{x}_0) \propto q(\mathbf{x}_t\mid\mathbf{x}_{t-1})\cdot q(\mathbf{x}_{t-1}\mid\mathbf{x}_0)\) 展开指数项，对 \(\mathbf{x}_{t-1}\) 配方即得。

1.4 ELBO 推导：为什么能这样训练？

直接计算 \(\log p_\theta(\mathbf{x}_0)\) 需要对所有可能的噪声路径积分，不可行。利用 Jensen 不等式引入变分下界（ELBO）[1]：

1

对任意分布 \(q\)，Jensen 不等式给出：

$$\log p_\theta(\mathbf{x}_0) \;\geq\; \mathbb{E}_q\!\left[\log\frac{p_\theta(\mathbf{x}_{0:T})}{q(\mathbf{x}_{1:T}\mid\mathbf{x}_0)}\right] \;:=\; -L_\text{VLB}$$
2

展开联合分布，利用马尔可夫性质，ELBO 分解为三项：

$$L_\text{VLB} = \underbrace{D_\text{KL}\!\left(q(\mathbf{x}_T\mid\mathbf{x}_0)\,\|\,p(\mathbf{x}_T)\right)}_{L_T,\ \text{无参数，丢弃}} + \sum_{t=2}^T \underbrace{D_\text{KL}\!\left(q(\mathbf{x}_{t-1}\mid\mathbf{x}_t,\mathbf{x}_0)\,\|\,p_\theta(\mathbf{x}_{t-1}\mid\mathbf{x}_t)\right)}_{L_{t-1},\ \text{核心训练项}} - \underbrace{\log p_\theta(\mathbf{x}_0\mid\mathbf{x}_1)}_{L_0,\ \text{重建项}}$$
3

两个均值不同、方差相同的高斯分布之间的 KL 有封闭解。令 \(p_\theta\) 的均值参数化为 \(\boldsymbol{\mu}_\theta(\mathbf{x}_t,t)\)，则 \(L_{t-1}\) 简化为：

$$L_{t-1} = \mathbb{E}\!\left[\frac{1}{2\tilde{\beta}_t}\left\|\tilde{\boldsymbol{\mu}}_t(\mathbf{x}_t,\mathbf{x}_0) - \boldsymbol{\mu}_\theta(\mathbf{x}_t,t)\right\|^2\right] + \text{const}$$
4

将 \(\mathbf{x}_0 = \frac{1}{\sqrt{\bar{\alpha}_t}}\!\left(\mathbf{x}_t - \sqrt{1-\bar{\alpha}_t}\,\boldsymbol{\epsilon}\right)\) 代入 \(\tilde{\boldsymbol{\mu}}_t\)，整理得：

$$\tilde{\boldsymbol{\mu}}_t = \frac{1}{\sqrt{\alpha_t}}\!\left(\mathbf{x}_t - \frac{1-\alpha_t}{\sqrt{1-\bar{\alpha}_t}}\,\boldsymbol{\epsilon}\right)$$

若网络预测 \(\boldsymbol{\epsilon}_\theta(\mathbf{x}_t,t)\) 并令 \(\boldsymbol{\mu}_\theta\) 采用相同结构，则 \(L_{t-1} \propto \|\boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta\|^2\)。

5

Ho et al. 发现去掉时间步权重因子，使用简单 MSE 效果更佳。最终训练目标为：

DDPM 核心训练目标 \(L_\text{simple}\) $$L_\text{simple} = \mathbb{E}_{t,\,\mathbf{x}_0,\,\boldsymbol{\epsilon}}\!\left[\left\|\boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta\!\left(\sqrt{\bar{\alpha}_t}\,\mathbf{x}_0 + \sqrt{1-\bar{\alpha}_t}\,\boldsymbol{\epsilon},\; t\right)\right\|^2\right]$$

随机选时间步 \(t\)，用封闭式加噪得到 \(\mathbf{x}_t\)，让网络预测所加的噪声 \(\boldsymbol{\epsilon}\)。

1.5 三种等价预测目标

DDPM 默认预测噪声 \(\boldsymbol{\epsilon}\)，但与另外两种目标完全等价，可按需选择。

预测目标含义转换关系常见场景
\(\boldsymbol{\epsilon}\)（噪声）预测所加入的噪声基准定义DDPM 默认，大多数图像生成
\(\mathbf{x}_0\)（干净图）直接预测原始图像\(\mathbf{x}_0 = (\mathbf{x}_t - \sqrt{1-\bar\alpha_t}\,\boldsymbol\epsilon)/\sqrt{\bar\alpha_t}\)低噪声步骤有时更稳定
\(\nabla_x\log q\)（分数）噪声概率密度的梯度\(\text{score} = -\boldsymbol\epsilon/\sqrt{1-\bar\alpha_t}\)理论分析，连接 score matching
\(\mathbf{v}\)（速度）Flow Matching 速度向量\(\mathbf{v} = \sqrt{\bar\alpha_t}\,\boldsymbol\epsilon - \sqrt{1-\bar\alpha_t}\,\mathbf{x}_0\)SD v-prediction，Flow Matching

1.6 采样（祖先采样）

训练完成后，从纯噪声出发，对 \(t = T, T-1, \ldots, 1\) 依次执行：

DDPM 祖先采样 $$\mathbf{x}_{t-1} = \frac{1}{\sqrt{\alpha_t}}\!\left(\mathbf{x}_t - \frac{1-\alpha_t}{\sqrt{1-\bar{\alpha}_t}}\,\boldsymbol{\epsilon}_\theta(\mathbf{x}_t,t)\right) + \sigma_t\,\mathbf{z}, \quad \mathbf{z}\sim\mathcal{N}(0,\mathbf{I})$$

其中 \(\sigma_t = \sqrt{\tilde\beta_t}\) 是后验标准差，\(t>1\) 时才加随机项。

⚠️ 主要瓶颈 DDPM 需要约 1000 步串行推理，在 V100 上生成一张 256×256 图像约需 30 秒。这是最大的实际限制，DDIM 专门来解决它。
算法二 · Song et al. 2020 [2]

DDIM 提速 10–50 倍

去噪扩散隐式模型——不改变训练，只改变采样方式，把 1000 步压缩到 10–50 步，并使过程变为确定性的。

2.1 马尔可夫结构不是必须的

DDPM 的前向过程是马尔可夫链，每步只依赖上一步。但 Song et al. 指出，训练目标 \(L_\text{simple}\) 只用到了边际分布 \(q(\mathbf{x}_t\mid\mathbf{x}_0)\)，而不是联合链。只要边际分布不变，训练目标就完全一样——因此可以找到一个更大的前向过程族，保持同样的边际，但允许确定性、可跳步的采样方式。

2.2 DDIM 更新公式推导

给定当前 \(\mathbf{x}_t\)，先估计干净图像：

第一步：估计 \(\mathbf{x}_0\) $$\hat{\mathbf{x}}_0 = \frac{\mathbf{x}_t - \sqrt{1-\bar{\alpha}_t}\;\boldsymbol{\epsilon}_\theta(\mathbf{x}_t,t)}{\sqrt{\bar{\alpha}_t}}$$

这相当于"给定当前噪声图，猜测原始干净图是什么"。

第二步：DDIM 一步更新 $$\mathbf{x}_{t-1} = \sqrt{\bar{\alpha}_{t-1}}\,\hat{\mathbf{x}}_0 + \sqrt{1-\bar{\alpha}_{t-1}-\sigma_t^2}\;\boldsymbol{\epsilon}_\theta(\mathbf{x}_t,t) + \sigma_t\,\mathbf{z}$$

\(\sigma_t\) 是可调的随机性控制参数，\(\mathbf{z}\sim\mathcal{N}(0,\mathbf{I})\)。

🔀 \(\sigma_t > 0\)（随机版本）
每步加入随机噪声，退化为类似 DDPM 的随机采样。当 \(\sigma_t = \sqrt{\tilde\beta_t}\) 时完全等价于 DDPM。
➡️ \(\sigma_t = 0\)（确定性版本）
纯确定性 ODE，固定 \(\mathbf{x}_T\) 就固定整条生成路径。支持跳步采样、语义插值和精确图像反演。

2.3 确定性版本的三大独特能力

当 \(\sigma_t=0\) 时，DDIM 在连续步长极限下对应概率流 ODE（probability flow ODE）：

概率流 ODE（连续极限） $$\mathrm{d}\mathbf{x} = \left[f(\mathbf{x},t) - \tfrac{1}{2}g(t)^2\,\nabla_{\mathbf{x}}\log p_t(\mathbf{x})\right]\mathrm{d}t$$

其中 \(\nabla_{\mathbf{x}}\log p_t(\mathbf{x}) \approx -\boldsymbol{\epsilon}_\theta(\mathbf{x},t)/\sqrt{1-\bar\alpha_t}\)。可以用大步长积分器（如 PLMS、DPM-Solver）高效求解。

✅ DDIM 的三大独特能力
  1. 快速采样：10–50 步即可达到接近 1000 步的质量。
  2. 语义插值：在噪声空间对两个 \(\mathbf{x}_T\) 线性插值，生成图像在语义上平滑过渡。
  3. 精确反演：给定真实图像，可反向积分 ODE 得到对应的 \(\mathbf{x}_T\)，用于图像编辑。
算法三 · Ho & Salimans 2022 [3]

CFG（无分类器引导）可控生成

Classifier-Free Guidance 给生成过程添加一个方向感旋钮，控制生成图像和文本条件的匹配程度，且不需要任何额外的分类器。

3.1 为什么条件模型会偷懒

训练条件扩散模型 \(p_\theta(\mathbf{x}\mid c)\) 时（\(c\) 是文本或类别），模型往往倾向于忽略条件 \(c\)：即使不看提示词，仅凭数据先验也能得到不错的损失。结果是生成的图像虽然多样，却与提示词关联松散。

3.2 前身：分类器引导

Dhariwal & Nichol（2021）用贝叶斯公式对条件分布做梯度分解 [8]：

贝叶斯梯度分解 $$\nabla_{\mathbf{x}}\log p(\mathbf{x}\mid y) = \underbrace{\nabla_{\mathbf{x}}\log p(\mathbf{x})}_{\text{无条件分数}} + \underbrace{\nabla_{\mathbf{x}}\log p(y\mid\mathbf{x})}_{\text{分类器梯度}}$$

用一个在噪声图像上训练的分类器 \(p_\phi(y\mid\mathbf{x}_t)\) 提供第二项，并放大 \(w\) 倍。缺点是需要额外训练噪声分类器，实现复杂。

3.3 CFG 的关键洞察

将贝叶斯公式改写，分类器梯度可以用条件预测与无条件预测的差值来隐式替代。

1

将贝叶斯公式反向运用：

$$\nabla_{\mathbf{x}}\log p(y\mid\mathbf{x}_t) = \nabla_{\mathbf{x}}\log p(\mathbf{x}_t\mid y) - \nabla_{\mathbf{x}}\log p(\mathbf{x}_t)$$
2

用分数与噪声预测的等价关系（\(\text{score} = -\boldsymbol\epsilon/\sqrt{1-\bar\alpha_t}\)）代换两边：

$$\text{隐式分类器梯度} \;\propto\; \boldsymbol{\epsilon}_\theta(\mathbf{x}_t\mid c) - \boldsymbol{\epsilon}_\theta(\mathbf{x}_t)$$
3

代入放大系数 \(w\)，得到 CFG 公式。

CFG 核心公式 $$\tilde{\boldsymbol{\epsilon}} = \boldsymbol{\epsilon}_\theta(\mathbf{x}_t,\,\varnothing) + w\cdot\bigl(\boldsymbol{\epsilon}_\theta(\mathbf{x}_t,\,c) - \boldsymbol{\epsilon}_\theta(\mathbf{x}_t,\,\varnothing)\bigr)$$

\(\boldsymbol{\epsilon}_\theta(\mathbf{x}_t,c)\) 为有条件预测，\(\boldsymbol{\epsilon}_\theta(\mathbf{x}_t,\varnothing)\) 为无条件预测，\(w\) 为引导强度。\(w=0\) 时退化为纯无条件生成，\(w\) 越大越贴合提示词但多样性越低。

引导强度 \(w\)效果典型使用场景
\(w=0\)纯无条件生成，忽略文本多样性探索
\(w=1\)普通条件生成，无放大自然遵从提示词
\(w\approx 3\text{–}8\)强调条件，牺牲多样性SD 默认区间
\(w>10\)过度引导，出现饱和伪影通常避免
✅ 训练技巧：null-token 随机丢弃 CFG 只需训练一个模型：每个训练样本以概率约 \(10\%\text{–}20\%\) 将条件 \(c\) 替换为空 token \(\varnothing\)，其余时间正常使用条件。模型自动学会两种模式。推理时每步需要两次前向传播（有条件 + 无条件）并线性组合。
算法四 · Rombach et al. 2022 [4]

LDM + VAE 高效

潜在扩散模型，将扩散过程从像素空间移入压缩潜变量空间，计算量降低约一个数量级，同时通过交叉注意力支持文本条件生成——这是 Stable Diffusion 的核心架构。

4.1 像素空间扩散为什么昂贵

在 512×512 图像上直接做扩散，U-Net 每次前向传播要处理约 786000 个像素值，且每个采样步骤都需要一次。大量算力花在了感知不敏感的高频细节上。Rombach et al. 指出，先将图像压缩到紧凑的潜在表示，再在潜在空间做扩散，可以在几乎不损失感知质量的前提下大幅降低计算量 [4]。

4.2 第一阶段：训练 VAE

变分自编码器（VAE）由编码器 \(\mathcal{E}_\phi\) 和解码器 \(\mathcal{D}_\theta\) 组成。编码器不输出单一向量，而是输出潜变量的分布参数：

VAE 架构 $$q_\phi(\mathbf{z}\mid\mathbf{x}) = \mathcal{N}\!\left(\mathbf{z};\; \boldsymbol{\mu}_\phi(\mathbf{x}),\; \boldsymbol{\sigma}_\phi(\mathbf{x})^2\mathbf{I}\right)$$ $$\mathbf{z} = \boldsymbol{\mu}_\phi(\mathbf{x}) + \boldsymbol{\sigma}_\phi(\mathbf{x})\odot\boldsymbol{\epsilon}, \quad \boldsymbol{\epsilon}\sim\mathcal{N}(0,\mathbf{I}) \quad\text{（重参数化技巧）}$$
💡 重参数化技巧的必要性 若直接对 \(\mathbf{z}\sim\mathcal{N}(\boldsymbol\mu,\boldsymbol\sigma^2\mathbf{I})\) 采样，梯度无法通过随机采样操作传播。将随机性分离出来写成 \(\mathbf{z}=\boldsymbol\mu+\boldsymbol\sigma\odot\boldsymbol\epsilon\)，随机项 \(\boldsymbol\epsilon\) 成为外部随机量（固定），梯度可以正常通过 \(\boldsymbol\mu\) 和 \(\boldsymbol\sigma\) 反向传播。这是 VAE 能用梯度下降训练的关键所在。

4.3 VAE 的 ELBO

1

目标是最大化数据似然的变分下界：

$$L_\text{VAE} = \underbrace{\mathbb{E}_{q_\phi(\mathbf{z}\mid\mathbf{x})}\!\left[\log p_\theta(\mathbf{x}\mid\mathbf{z})\right]}_{\text{重建损失}} - \underbrace{D_\text{KL}\!\left(q_\phi(\mathbf{z}\mid\mathbf{x})\,\|\,p(\mathbf{z})\right)}_{\text{KL 正则，逼近先验 }\mathcal{N}(0,\mathbf{I})}$$
2

高斯 KL 散度有解析式：

$$D_\text{KL}\!\left(\mathcal{N}(\boldsymbol{\mu},\boldsymbol{\sigma}^2\mathbf{I})\,\|\,\mathcal{N}(0,\mathbf{I})\right) = \frac{1}{2}\sum_i\!\left(\mu_i^2 + \sigma_i^2 - \log\sigma_i^2 - 1\right)$$

KL 项鼓励 \(\boldsymbol\mu\to\mathbf{0},\,\boldsymbol\sigma\to\mathbf{1}\)（正则化），重建项鼓励准确还原（表达力），两者形成经典的正则-表达力权衡。

3

LDM 的 VAE 在像素重建基础上还加入了感知损失和对抗损失：

$$L_\text{LDM-VAE} = L_\text{recon} + \lambda_\text{KL}\,L_\text{KL} + \lambda_\text{perc}\,L_\text{LPIPS} + \lambda_\text{adv}\,L_\text{adversarial}$$

LPIPS 感知损失用 VGG 特征衡量高频质量；对抗损失（patch-based 判别器）保证图像真实感。\(\lambda_\text{KL}\) 极小（约 \(10^{-6}\)），保留足够的潜在空间表达力。

4.4 第二阶段：在潜在空间做扩散

VAE 训练完成后冻结，扩散过程完全在压缩潜变量 \(\mathbf{z}\) 上进行：

LDM 训练目标 $$L_\text{LDM} = \mathbb{E}_{\mathbf{z}=\mathcal{E}(\mathbf{x}),\,\boldsymbol{\epsilon},\,t}\!\left[\left\|\boldsymbol{\epsilon} - \boldsymbol{\epsilon}_\theta\!\left(\mathbf{z}_t,\;t,\;\tau_\theta(y)\right)\right\|^2\right]$$

\(\mathbf{z}_t = \sqrt{\bar\alpha_t}\,\mathbf{z} + \sqrt{1-\bar\alpha_t}\,\boldsymbol\epsilon\) 是在潜变量上加噪；\(\tau_\theta(y)\) 是文本或其他条件的编码，通过交叉注意力注入 U-Net。推理时从 \(\mathbf{z}_T\sim\mathcal{N}(0,\mathbf{I})\) 去噪得到 \(\hat\mathbf{z}_0\)，再解码 \(\hat\mathbf{x} = \mathcal{D}(\hat\mathbf{z}_0)\)。

4.5 文本条件注入：交叉注意力

文本 prompt 通过 CLIP 或 T5 编码为 token 序列，再通过交叉注意力层注入 U-Net 中间层：

交叉注意力（Cross-Attention） $$\mathbf{Q} = W_Q\,\varphi(\mathbf{z}), \quad \mathbf{K} = W_K\,\tau_\theta(y), \quad \mathbf{V} = W_V\,\tau_\theta(y)$$ $$\text{Attention}(\mathbf{Q},\mathbf{K},\mathbf{V}) = \text{softmax}\!\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d}}\right)\mathbf{V}$$

图像潜在特征提供 Query，文本编码提供 Key 和 Value。这样，每个空间位置的图像特征都可以关注最相关的文本 token。

✅ LDM 的效率收益 8× 空间下采样（64×64 对比 512×512 像素面积），在相同模型结构下，U-Net 每步计算量降低约 64 倍。这就是 Stable Diffusion 能在消费级 GPU 上运行的根本原因。
算法五 · Peebles & Xie 2023 [5]

DiT（扩散 Transformer）可扩展

用 Transformer 替换 U-Net 骨干网络，获得可预测的扩展律。现在 Sora、Stable Diffusion 3、CogVideoX 等主流系统都以此为架构基础。

5.1 为什么 U-Net 需要被替换

U-Net 有很强的空间归纳偏置（卷积、多尺度跳接），在数据量少时表现很好。但它的扩展律不够清晰：加宽加深 U-Net，FID 的下降曲线不规律。LLM 和 VLM 的成功证明 Transformer 有可预测的幂律扩展，Peebles & Xie 的贡献就是将这种扩展律引入图像生成。

5.2 把图像变成 token 序列

DiT 在 LDM 的潜在空间中运行。对 256×256 图像，VAE 得到 \(32\times32\times4\) 的潜变量，然后类似 ViT 切成 patch：

Patchify 参数 $$\text{patch 大小} \;p \in \{2,4,8\}, \quad \text{token 数} \;T = \left(\frac{\text{latent\_size}}{p}\right)^2$$ $$\text{每个 token 维度} = p\times p\times C \;\xrightarrow{\text{线性投影}}\; d_\text{model}$$ $$\text{计算量 Gflops} \propto T \propto p^{-2}, \quad \text{即 } p \text{ 减半则计算量翻四倍}$$

更小的 patch 意味着更多 token，更多计算量，也带来更好的生成质量。DiT-XL/2 表示最大模型、patch 大小为 2。

5.3 关键创新：adaLN-Zero 条件注入

时间步 \(t\) 和类别/文本条件 \(c\) 通过自适应层归一化（adaLN）注入每个 Transformer 块。论文对比了四种方案：

方案机制400K 步 FID↓额外计算量
In-context把 \(t,c\) 作为额外 token 拼接119.9极小
Cross-Attention\(t,c\) 作为 K/V45.9中等
adaLN由 \(t,c\) 预测 LayerNorm 的 scale/shift47.9极小
adaLN-ZeroadaLN + 零初始化残差门控23.5极小
adaLN-Zero 计算流程 $$\mathbf{c}_\text{emb} = \text{MLP}(\mathbf{t}_\text{emb} + \mathbf{c}_\text{emb})$$ $$(\gamma_1,\beta_1,\alpha_1,\gamma_2,\beta_2,\alpha_2) = \text{MLP}_\text{mod}(\mathbf{c}_\text{emb})$$ $$\mathbf{h}' = \mathbf{x} + \alpha_1\cdot\text{Attention}\!\left(\text{adaLN}(\mathbf{x};\gamma_1,\beta_1)\right)$$ $$\mathbf{h} = \mathbf{h}' + \alpha_2\cdot\text{MLP}\!\left(\text{adaLN}(\mathbf{h}';\gamma_2,\beta_2)\right)$$

其中 \(\alpha_1,\alpha_2\) 初始化为 0（零初始化），使每个 DiT Block 一开始等价于恒等变换，大幅稳定大规模训练。

5.4 扩展律：为什么这件事如此重要

Peebles & Xie 的核心实验结果表明，以 Gflops 为横轴、FID 为纵轴，DiT 呈现出近乎完美的单调幂律下降关系。最大的 DiT-XL/2（119 Gflops/步）在 ImageNet 256×256 类别条件生成上达到 FID = 2.27，超过计算量高达 742 Gflops 的 ADM-U [5]。这种规律性让研究者能够事先预测更大模型的效果，正是 Sora 等系统敢于投入大量计算资源的信心来源。

🔧 U-Net 仍然适合的场景
数据量少、计算预算有限；需要强空间归纳偏置（医学图像等）；已有成熟工具链和社区支持。
🚀 DiT 更适合的场景
大规模数据加计算（scaling regime）；需要与 LLM/VLM 共享架构；视频或多模态序列；需要可预测扩展律的工业级系统。
算法六 · Lipman et al. 2022 [6]

Flow Matching 更快更好

流匹配以直线传输路径替代 DDPM 的曲线加噪路径，训练目标更简单，采样所需步骤更少，且从数学上严格包含扩散模型作为特例。视频和机器人动作生成领域几乎已全面切换到这一范式。

6.1 DDPM 的路径为什么是弯的

DDPM 的前向过程是方差保持的高斯路径，在分布空间里形成曲线。ODE 积分器沿曲线行进时必须用小步长以控制截断误差——这就是需要上千步的根本原因。

Flow Matching 的核心想法很直接：如果能让运输路径尽可能笔直，积分步骤自然大幅减少。

6.2 从连续归一化流（CNF）出发

Flow Matching 建立在连续归一化流的框架上。给定时变向量场 \(v_t(\mathbf{x})\)，定义 ODE：

流 ODE $$\frac{\mathrm{d}\boldsymbol{\phi}_t(\mathbf{x})}{\mathrm{d}t} = v_t\!\left(\boldsymbol{\phi}_t(\mathbf{x})\right), \quad \boldsymbol{\phi}_0(\mathbf{x}) = \mathbf{x}$$

流 \(\boldsymbol\phi_t\) 将 \(t=0\) 时的噪声分布 \(p_0=\mathcal{N}(0,\mathbf{I})\) 运输到 \(t=1\) 时的数据分布 \(p_1=p_\text{data}\)。分布演化满足连续性方程 \(\partial_t p_t = -\text{div}(v_t\,p_t)\)。

6.3 为什么直接匹配边际向量场不可行

看起来只需要回归那个"正确的"向量场就行：

边际 FM 目标（不可行） $$L_\text{FM}(\theta) = \mathbb{E}_{t,\,\mathbf{x}\sim p_t}\!\left[\|v_\theta(\mathbf{x},t) - u_t(\mathbf{x})\|^2\right]$$

问题在于边际向量场 \(u_t(\mathbf{x}) = \int u_t(\mathbf{x}\mid\mathbf{x}_1)\,\frac{p_t(\mathbf{x}\mid\mathbf{x}_1)\,p_\text{data}(\mathbf{x}_1)}{p_t(\mathbf{x})}\,\mathrm{d}\mathbf{x}_1\) 需要对整个数据分布积分，计算不可行。

6.4 条件流匹配（CFM）定理

Lipman et al. 的核心定理（Theorem 2）给出了优雅的解法 [6]：

CFM 等价定理 $$L_\text{CFM}(\theta) = \mathbb{E}_{t,\,\mathbf{x}_1\sim p_\text{data},\,\mathbf{x}\sim p_t(\mathbf{x}\mid\mathbf{x}_1)}\!\left[\|v_\theta(\mathbf{x},t) - u_t(\mathbf{x}\mid\mathbf{x}_1)\|^2\right]$$ $$\nabla_\theta L_\text{FM} = \nabla_\theta L_\text{CFM} \quad \text{（梯度完全相等）}$$

条件向量场 \(u_t(\mathbf{x}\mid\mathbf{x}_1)\) 只依赖单个数据样本 \(\mathbf{x}_1\)，计算简单。两个目标的梯度相等，因此优化 CFM 就是优化 FM。

6.5 最优传输线性路径：最简洁的特例

1

选择从噪声到数据的线性插值路径：

$$\mathbf{x}_t = (1-t)\,\mathbf{x}_0 + t\,\mathbf{x}_1, \quad t\in[0,1]$$

\(\mathbf{x}_0\sim\mathcal{N}(0,\mathbf{I})\)（噪声），\(\mathbf{x}_1\sim p_\text{data}\)（真实数据）。

2

对 \(\mathbf{x}_t\) 关于 \(t\) 求导，得到条件速度场：

$$u_t(\mathbf{x}_t\mid\mathbf{x}_1) = \frac{\mathrm{d}\mathbf{x}_t}{\mathrm{d}t} = \mathbf{x}_1 - \mathbf{x}_0$$

这个速度是常数！不依赖 \(t\)，也不依赖 \(\mathbf{x}_t\) 本身，只依赖端点 \(\mathbf{x}_0,\mathbf{x}_1\)。这正是训练更稳定的原因。

3

代入 CFM 损失，得到极简形式：

Flow Matching 核心训练目标（OT-CFM） $$L_\text{FM} = \mathbb{E}_{t,\,\mathbf{x}_0\sim\mathcal{N}(0,\mathbf{I}),\,\mathbf{x}_1\sim p_\text{data}}\!\left[\left\|v_\theta\!\left((1-t)\mathbf{x}_0+t\mathbf{x}_1,\;t\right) - \left(\mathbf{x}_1-\mathbf{x}_0\right)\right\|^2\right]$$

采样一对噪声和数据，插值得到中间点，让网络预测从起点到终点的方向向量。目标是常数，训练方差更低，过程更稳定。

对比维度DDPMFlow Matching（OT-CFM）
路径类型方差保持高斯路径（曲线）线性最优传输路径（直线）
训练目标预测噪声 \(\boldsymbol\epsilon\)预测速度向量 \(\mathbf{x}_1-\mathbf{x}_0\)（常数）
目标依赖 \(t\)是（通过 \(\sqrt{\bar\alpha_t}\) 等权重）否
典型采样步数500–1000 步10–50 步
训练稳定性需仔细调节 \(\beta\) 调度表更稳定，梯度方差更小
理论关系FM 的一个特例严格包含 DDPM
Flow Matching 采样（Euler 方法） $$\mathbf{x}_0\sim\mathcal{N}(0,\mathbf{I}), \quad \mathbf{x}_{t+\Delta} = \mathbf{x}_t + \Delta\cdot v_\theta(\mathbf{x}_t,t), \quad t: 0\to 1$$

路径越直，步长 \(\Delta\) 可以越大，误差越小。理论上直线路径一步即可，实际中用 10–20 步已能得到高质量结果。

算法七 · CogVideoX 2024 [7] · Open-Sora 2024 [9]

Video DiT 视频生成

3D 因果 VAE + 时空 patch 化 + 文本注入，将前六章的所有技术堆叠到时间维度上，构成 Sora、CogVideoX、Open-Sora 的共同架构原理。

7.1 视频生成的三个额外挑战

  • 时序一致性：相邻帧必须语义和运动连贯，不能"闪烁"或出现物体突变。
  • 巨大序列长度：5 秒 24fps 视频有 120 帧，若直接 patch 化，token 数是等尺寸图像的 120 倍。
  • 时空压缩必要性：必须同时在空间和时间维度压缩，否则模型无法在合理计算预算内训练。

7.2 第一步：3D 因果 VAE

将 2D 空间 VAE 升级为 3D 卷积 VAE，同时压缩空间和时间。CogVideoX 的 3D VAE 实现了 4× 时间压缩加 8× 空间压缩 [7]：

CogVideoX 3D VAE 压缩率 $$\text{输入: } T\times H\times W\times 3 \;\xrightarrow{\text{3D causal VAE}}\; \frac{T}{4}\times\frac{H}{8}\times\frac{W}{8}\times 16$$

总 token 数减少 256 倍，与仅做空间压缩（64倍）相比额外压缩了 4 倍时间维度。CogVideoX 选取的变体 B 在 PSNR（28.7）和抖动评分（86.3）上的综合表现最佳。

💡 因果 3D 卷积的设计逻辑 普通 3D 卷积会让第 \(t\) 帧"看到"未来帧，破坏时序因果性。因果 3D 卷积把所有时间维度的填充放在序列开头，确保每帧只依赖当前和历史帧。这样第一帧可以作为图生视频的条件输入，训练和推理的逻辑也因此统一。

7.3 第二步：时空 Patch 化

将 3D 潜变量沿空间和时间维度切割为时空 patch，每个 patch 经线性投影成为一个 token。Sora 将此类比为 LLM 的 token 概念：视觉 patch 在视频数据中的地位，等同于 token 在文本数据中的地位 [10]。

时空 Patch 化 $$\text{3D 潜变量: } (T',H',W',C) \;\xrightarrow{\text{切割 }(p_t,p_h,p_w)}\; N_\text{tok}\times(p_t p_h p_w C) \;\xrightarrow{\text{线性投影}}\; N_\text{tok}\times d_\text{model}$$ $$N_\text{tok} = \frac{T'}{p_t}\cdot\frac{H'}{p_h}\cdot\frac{W'}{p_w}$$

位置编码采用 3D RoPE（CogVideoX-5B）或 3D 正弦余弦编码（CogVideoX-2B）。

7.4 文本条件注入：两种主流方式

📌 交叉注意力（Open-Sora 等）
视觉 token 作 Query，T5 文本编码作 Key/Value，通过交叉注意力层注入。结构清晰，与图像 LDM 完全兼容。
🔗 拼接全自注意力（CogVideoX）
将 T5 文本嵌入和视频潜变量沿序列维度拼接后输入完整自注意力。CogVideoX 还引入 Expert AdaLN，为文本 token 和视觉 token 分别配置独立的 LayerNorm 参数。

7.5 训练目标：整流流

现代 Video DiT（CogVideoX、Open-Sora 1.2+、SD3）几乎全部采用整流流（Rectified Flow，即 Flow Matching）训练。Open-Sora 从 SD3 借鉴的关键细节 [9]：

Open-Sora 整流流训练细节 $$L = \mathbb{E}\!\left[\left\|v_\theta(\mathbf{z}_t,t,c) - (\mathbf{z}_1-\mathbf{z}_0)\right\|^2\right]$$

附加细节：时间步 \(t\) 采用 logit-normal 采样（偏向中间噪声级别）；高分辨率/长视频使用更大的时间步偏移（因为更难去噪）；每个通道独立归一化对整流流训练至关重要。

✅ Video DiT 是前六章的总装 VAE（压缩像素到潜变量）× 3D 扩展（加时间维度）+ DiT 架构（Transformer 骨干）× Flow Matching（训练目标）× CFG（文本引导）+ DDIM 风格 ODE 采样（推理加速）——六项技术缺一不可。
应用 · 机器人控制

桥梁：DiT + Flow Matching 到机器人动作生成

视频生成技术为何能直接用于控制机器人？π0 和 World Action Model 如何用流匹配生成连续动作序列？

8.1 动作序列和图像像素的本质同构

扩散和流匹配学到的是：把噪声分布运输到目标分布。在图像生成里，目标是像素的联合分布；在机器人控制里，只需将目标换为未来动作序列的分布，架构和损失函数几乎原封不动。

机器人任务天然需要对多模态分布建模——同一个"把杯子放到托盘"的任务，有无数合法的手腕轨迹。流匹配比简单的 MSE 回归更适合这种情况，因为后者只能学到众多合法动作的均值，导致机器人犹豫不决或动作生硬。

💡 结构对比 图像生成：从噪声图像出发，条件为文本 prompt，最终得到真实图像。
动作生成：从噪声动作序列出发，条件为当前视觉观测和任务语言，最终得到真实动作序列。
两者的数学形式完全一样，只是数据从像素向量换成了关节角度向量。

8.2 π0 的流匹配动作策略 [11]

π0 将一个 300M 参数的动作专家附接到 3B 参数的 PaliGemma VLM 上，总参数量 3.3B。两个专家通过共享自注意力交互，图像和语言路由到 VLM 骨干，关节状态和噪声动作路由到动作专家。

π0 动作流匹配训练目标 $$\mathbf{A}_t^\tau = \tau\cdot\mathbf{A}_t + (1-\tau)\cdot\boldsymbol{\epsilon}, \quad \boldsymbol{\epsilon}\sim\mathcal{N}(0,\mathbf{I}), \quad \tau\in[0,1]$$ $$L(\theta) = \mathbb{E}_{\tau,t}\!\left[\left\|v_\theta\!\left(\mathbf{A}_t^\tau,\,\mathbf{o}_t\right) - \left(\mathbf{A}_t - \boldsymbol{\epsilon}\right)\right\|^2\right]$$

\(\mathbf{A}_t = [a_t,\ldots,a_{t+H-1}]\) 是 \(H=50\) 步的动作块；\(\mathbf{o}_t = \{\text{图像},\text{语言},\text{关节状态}\}\) 为条件（通过 VLM 编码后 KV 缓存）；\(\tau=0\) 时为纯噪声，\(\tau=1\) 时为干净动作。
时间步 \(\tau\sim\text{Beta}((s-\tau)/s;\,1.5,1)\) 偏向低 \(\tau\)（高噪声），\(s=0.999\)。推理时用 10 步 Euler 积分，条件的 KV 缓存，控制频率可达 50 Hz。

8.3 World Action Model（WAM）

WAM 走得更远：用单个统一的视频-动作扩散 Transformer，同时预测未来的视觉潜变量（世界会怎样变化）和机器人动作（我应该怎么做）[12]。NVIDIA 将 WAM 描述为：把感知、世界建模和动作生成整合到单一架构中，让机器人能够直接从视频数据的隐式物理知识中获益。

✅ DiT + Flow Matching 成为机器人控制标配的四个原因
  1. 统一 token 接口：Transformer 天然处理异构序列（图像 patch、语言 token、动作向量）。
  2. 多模态分布建模：FM 能准确建模同一任务下的多种合法动作，而非输出均值。
  3. 少步快速采样：FM 的直线路径使 10 步 Euler 积分即可满足 50 Hz 实时控制需求。
  4. 从视频预训练迁移：相同骨干权重可从视频生成微调到机器人控制，数据效率高。

统一视角：所有方法都是概率传输的不同实例

七个方法及其变体，都可以用同一个框架描述：给定简单先验 \(p_0\) 和数据分布 \(p_1\)，学习时变向量场 \(v_\theta(\mathbf{x},t)\) 使对应的流将 \(p_0\) 运输到 \(p_1\)。不同方法的区别仅在于路径选择和预测参数化方式：

统一框架 $$\text{DDPM: 方差保持高斯路径} \;\longrightarrow\; \text{预测 } \boldsymbol{\epsilon}$$ $$\text{FM-OT: 线性最优传输路径} \;\longrightarrow\; \text{预测 } \mathbf{v} = \mathbf{x}_1 - \mathbf{x}_0$$ $$\text{Score: 与 DDPM 同路径} \;\longrightarrow\; \text{预测 } \nabla_{\mathbf{x}}\log p_t$$

三种目标线性等价：\(\mathbf{v} = \sqrt{\bar\alpha_t}\,\boldsymbol\epsilon - \sqrt{1-\bar\alpha_t}\,\mathbf{x}_0\)，\(\text{score} = -\boldsymbol\epsilon/\sqrt{1-\bar\alpha_t}\)。理解其中一个，其他两个可直接推导。

⚠️ 常见陷阱 时间方向约定不统一。 Flow Matching 文献中存在两种惯例：Lipman et al. 和 π0 用 \(t=0\) 表示噪声、\(t=1\) 表示数据；许多扩散论文则相反。复现代码时务必先确认方向，这是最常见的 bug 来源。

Sora 的内部细节是合理推断，非官方公开。 OpenAI 的 Sora 报告刻意保持高层次，架构细节是社区基于 CogVideoX 和 Open-Sora 等开源系统做出的推断。以 CogVideoX 和 Open-Sora 为权威开源参考。

CFG 引导强度没有万能值。 通常引用的 \(w\approx 7.5\) 是 SD 1.x 的经验设置，不同模型和任务下最优值差异显著，需要根据实际效果调整。
