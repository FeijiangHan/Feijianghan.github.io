# 1.Basic Overview

# 1.1 编译-链接

开始之前简介一下C/C++编译链接的过程：

对于一个源文件，编译器会读取它的所有头文件，并把它们的声明和定义加载进来，然后将源文件和头文件一起编译成目标文件(.o文件)。在链接阶段，链接器会把所有的目标文件和库文件(.a/.so)链接在一起，生成可执行文件。

我们自己程序编译后：得到.o文件

下载外部库后：得到.a或者.so文件（这都是linux环境下的，windows下应该是.lib文件）

.o文件是编译之后的目标文件，是可重定位的二进制文件，包含了编译之后的代码和数据。

.a文件和.so文件是静态库和动态库，它们都是已经编译好的库文件，可以被其他程序连接。

.a文件和.o文件都是二进制文件，它们都是编译后的中间文件。.a文件是静态库文件，它是由一系列.o文件打包成的文件，在链接阶段会把这些.o文件和其他.a文件或者.o文件链接到一起，生成一个可执行文件。而.o文件是单独的一个编译后的中间文件，它需要和其他的.o文件或者.a文件链接在一起才能生成可执行文件。

# 1.2静态链接和动态链接

静态链接是在编译时将链接库的二进制代码嵌入到可执行文件中，所以可执行文件的大小会增加。

.o和.o之间的链接是静态链接（一般情况） .o和.a之间的链接是静态链接 动态链接是在运行时加载链接库，所以可执行文件的大小会更小，但是需要额外的链接库文件（.so文件）。

.o和.so之间的链接则是动态链接。 我们下载下来的库一般都是静态库，即以.a结尾的文件，其是通过静态链接和其他.o文件链接到一起的。还有一种库下载以后lib里面是.so文件，其是通过动态链接和其他文件链接到一起的。

.a文件是静态库，在编译的时候会将静态库的代码直接嵌入到程序中，这样可以避免在运行时出现动态链接的问题，但是会增加程序的体积。 .so文件是动态库，在编译的时候不会将动态库的代码嵌入到程序中，而是在运行时进行动态链接，这样可以减小程序的体积，但是需要保证动态库在运行时能够被找到并正确链接。

# 1.3 CMake的作用

CMake是一个构建管理工具，它可以用来管理编译和链接过程。 在CMake中，你可以指定要编译的源文件、头文件、第三方库、编译选项等信息，然后自动生成编译和链接的脚本，最后生成可执行文件或者动态链接库。

它主要包含以下几部分信息：

项目信息，包括项目名称、版本号、语言等。 源代码文件的路径信息，如头文件和源文件的路径。 第三方库的路径信息，如链接库路径、库名称等。 编译选项，如编译器、编译标志等。 生成脚本，如生成可执行文件、动态链接库等。

# 2.Details

对于其他项目来说，上面的cmake改改就可以执行了，但是还是有一些细节需要记录。

# 2.1 project设置项目名

project(name)是CMake项目信息中的项目名称，它告诉CMake这个项目的名字是deadcode. 这个命令还可以指定项目的版本号和语言等信息，如 "project(nameVERSION 1.0 LANGUAGES CXX)"。通过project()函数设置项目名称后，变量PROJECT_NAME才会被赋值。

# 2.2 set变量赋值

我们可以使用set函数设置一个变量，其值为我们自己项目的源文件路径，下面这种方式是按照模块或者功能划分，先放所有的.h文件，再放所有的.cpp文件，这样可以方便维护和阅读。并且在每一个模块之间加上空行，这样可以更好的区分不同的模块。

# 2.4 include_directories和link_directories引入外部库

从github下载外部库后，里面应该会有include文件夹和lib文件夹。include文件夹中存放头文件，以.h结尾，里面是我们需要调用的API接口；lib文件夹存放链接库，一般是编译后的文件，以.a或.so结尾。

include_directories用来指定编译器搜索头文件的路径。当你在代码中使用#include <header.h>的方式来包含头文件时，编译器会在include_directories指定的路径中查找头文件。也就是说，当我们将其设置为我们下载库的include文件夹后，在我们的项目中我们可以直接以头文件的名字include外部库的头文件，不需要以路径的形式引入了。

link_directories用来指定编译器搜索库文件的路径。当你在代码中使用target_link_libraries()来链接库文件时，编译器会在link_directories指定的路径中查找库文件。也就是说，当我们将其设置为我们下载库的lib文件夹后，链接时就会在指定外部库的lib文件中寻找对于的.a静态库或者.so动态库。

##2.5 find_package引入线程库 find_package(Threads)是CMake中的一个指令，用于在系统中查找并加载线程库。它会查找线程库（通常是pthreads或Windows Threads）并设置一些CMake变量，以便在编译和链接时使用该库。

Threads::Threads是CMake中的一个目标名称，指向在find_package(Threads)中查找到的线程库。在链接编译好的可执行文件时，使用 target_link_libraries(${PROJECT_NAME} Threads::Threads) 可以将线程库链接在一起。这样可以确保你的程序能够正确地使用线程。

# 3.加载线程库

find_package (Threads)

# 2.6 target_link_libraries链接

前面的add_executable将我们自己的项目编译为.o文件，target_link_libraries 函数是将预先编译好的库（通常是.a或.so文件）链接到项目的.o文件，使得项目可以使用这些库中的函数和类。在这个例子中，libxed.a和Threads::Threads是链接到项目中的两个外部库， ${PROJECT_NAME} 是项目的名称。

#将第三方库链接在一起 target_link_libraries (${PROJECT_NAME} libxed.a Threads::Threads)

3.注意事项 静态库的名称一般是libxxx.a，动态库的一般名称是libxxx.so，不要随意修改库的名字，否则会出现找不到库文件的错误。 CMake配置文件中的一些语句可以适当调整顺序，但add_executable语句必须要在target_link_libraries语句的前面。 .lib文件是Windows平台下的静态链接库文件，它是由若干个.o文件通过静态链接器链接而成的。 .a文件是Linux和Unix系统下的静态链接库文件，它也是由若干个.o文件通过静态链接器链接而成的。 两者的区别在于，.lib文件是Windows平台下特有的静态链接库文件，而.a文件是Linux和Unix系统下的静态链接库文件。
